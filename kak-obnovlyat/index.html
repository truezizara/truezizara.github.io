Как обновлять сайт

Личная шпаргалка. Всё делается на сайте github.com, в репозитории truezizara.github.io. После любого сохранения (Commit) сайт обновляется сам за 1–2 минуты.

✏️ Добавить заметку о Японии

1. Открой папку japan/_posts → кнопка Add file → Create new file.

2. Имя файла: 2026-07-15-nazvanie-latinicej.md — сначала дата, потом короткое название латиницей, в конце .md.

3. В само поле вставь шаблон и пиши текст ниже трёх дефисов:

---
title: "Заголовок заметки по-русски"
---

Дальше обычный текст. Пустая строка — новый абзац.

4. Зелёная кнопка Commit changes. Всё — заметка на сайте, в списке на странице «Япония».

💬 Добавить пост в Блог

То же самое, но в папке blog/_posts. Скопировала пост из Телеграма → вставила под шапкой с title → Commit.

🗺 Добавить гайд

То же самое, папка guides/_posts. Гайд появится на странице «Япония» в секции «Гайды».

📷 Добавить фотоальбом

Два шага.

Шаг 1 — фотки. Открой папку photos → Add file → Upload files. Чтобы создать новую папку: Create new file, в поле имени набери kioto-2026/temp.md (слэш создаёт папку), закоммить, затем зайди в неё и загрузи фото через Upload files (а temp.md удали). Либо проще: Upload files и перетащи с компьютера сразу папку с фотографиями — она приедет вместе с именем.

Шаг 2 — карточка альбома. В папке _albums создай файл kioto-2026.md с шапкой:

---
title: "Киото весной"
date: 2026-04-10
type: film
folder: kioto-2026
---

type — это film (плёнка) или iphone. folder — имя папки с фотками из шага 1. Всё: альбом появился на странице «Фото» в нужной вкладке, лента собралась сама.

Обложка альбома — первое фото по алфавиту. Хочешь конкретное — назови файл cover.jpg.

Совет: перед загрузкой ужимай фото до 1600–2000 px по длинной стороне — быстрее грузится и место в репозитории не кончится.

📍 Обновить карту Японии

Экспортируй KML из Place Collector, переименуй файл в places.kml и загрузи в папку assets (Upload files — файл с тем же именем перезапишется). Точки на карте обновятся сами.

🖼 Вставить фото внутрь заметки

1. Загрузи картинку в папку assets.

2. В тексте заметки напиши строку: ![что на фото](/assets/imya-fajla.jpg)

🔧 Поправить опечатку где угодно

Найди файл → иконка карандаша ✏️ справа сверху → правишь → Commit changes.

⚠️ Что лучше не трогать

Папки _layouts, _includes, css, файл _config.yml — это внутренности. Если что-то в них захочется поменять — приходи ко мне с редакцией 🙂

🆘 Если сайт «сломался»

Чаще всего — из-за шапки файла: три дефиса сверху и снизу, а заголовок в кавычках. GitHub пришлёт письмо «Page build failed», если сборка не удалась — просто проверь последний изменённый файл. И помни: в git ничего не теряется, любую правку можно откатить.
